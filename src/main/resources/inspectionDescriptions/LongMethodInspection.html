Detect Code smell of "Long Method".


This inspection detect methods that are longer than the specified "method length".
Specify the allowable upper limit for "method length".
The "method length" here refers to the number of statement.


How to refactor
1. Go to description of "Long Parameter List" if parameter list is too long.
2. Execute refactoring of "Replace Temp with Query" if it is not easy to extract as another method because temporary variables are too many.
3. Execute refactoring of "Decompose Conditional" if there are complex conditional branches.
4. Execute refactoring of "Replace Method with Method Object" if parameters or temporary variables exist after you execute refactoring 1 and 2.
5. Execute refactoring of "Extract Method" if there are parts that can be combined by multiple methods.

コードスメル『長すぎるメソッド』を検出します。


このインスペクションは指定した『メソッドの長さ』より長いメソッドを検出します。
『メソッドの長さ』の許容上限を指定してください。
ここでの『メソッドの長さ』とはステートメントの数を指します。


リファクタリング方法
1.引数が多い場合は、『長すぎるパラメータリスト』の説明を見てください。
2.一時変数が多すぎるためにメソッドの抽出が困難な場合は、『問い合わせによる一時変数の置き換え』を行ってください。
3.複雑な条件式がある場合は、『条件記述の分解』を行ってください。
4.リファクタリング1や2を行った後、どうしても一時変数や引数が残ってしまった場合、『メソッドオブジェクトによるメソッドの置き換え』を行ってください。
5.複数のメソッドでまとめられる箇所がある場合、『メソッドの抽出』を行ってください。
